## 2. Protocol

- Разбиение:
    Метод: train_test_split с стратификацией
    Размер теста: 25% (3,000 объектов)
    Random state: 42
    Стратификация по target для сохранения распределения классов
- Подбор:
    Метод: GridSearchCV
    Валидация: StratifiedKFold с 5 фолдами
    Random state: 42
    Оптимизируемая метрика: ROC-AUC
    Данные для подбора: только тренировочная выборка (9,000 объектов)
- Метрики: accuracy, F1, ROC-AUC
    Accuracy: общая доля правильных предсказаний. Уместна как базовая метрика, но может быть обманчива при дисбалансе
    F1-score (binary): гармоническое среднее precision и recall. Критически важна при умеренном дисбалансе классов
    ROC-AUC: площадь под ROC-кривой. Оптимальна для бинарной классификации с дисбалансом. Оценивает способность модели ранжировать объекты. Основная метрика для сравнения моделей
